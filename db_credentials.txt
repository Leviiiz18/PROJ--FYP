# Kiara Academy - Database Index & Access Guide

All databases in this project use SQLite 3. This means there are NO usernames, passwords, or server addresses. The "credentials" are the local file paths.

---

### 1. Mock Exam Database (Restored & Rebuilt)
- **Purpose**: Stores student exam attempts, scores, and the PRE-PARSED textbook cache.
- **File Path**: `d:\code\kt\mock_test\exam_data.db`
- **Tables**:
    - `attempts`: Historical scores and feedback.
    - `textbook_cache`: The pre-parsed text from your PDFs for lightning-fast exam generation.

### 2. Student Progress & Session Data
- **Purpose**: Stores the main chat sessions and RAG history.
- **File Path**: `d:\code\kt\rag\db\student_data.db` (Verify if this is the correct path in the rag/ folder)

### 3. Vector Database (Classroom Retrieval)
- **Purpose**: ChromaDB/FAISS indexes for the real-time teacher chat.
- **File Path**: `d:\code\kt\vector_store\`

---

### 🛠️ How to View Data in a Text File
I have provided two ways to view your data:

**Method A: Automatic Export (Recommended)**
Run my provided script to dump EVERYTHING into a readable text file:
`.venv\Scripts\python d:\code\kt\mock_test\export_database.py`
This will create a `database_dump.txt` in your root folder.

**Method B: Manual SQL Inspection**
You can use the `sqlite3` command in your terminal:
1. `sqlite3 d:\code\kt\mock_test\exam_data.db`
2. `.headers on`
3. `.mode column`
4. `SELECT * FROM attempts;`
5. `.quit`

**Method C: GUI Tools**
Download "DB Browser for SQLite" (free) to view your tables like an Excel sheet.
